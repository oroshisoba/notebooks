# Copilot Instructions

このリポジトリでは、GitHub Copilot は日本のシステム開発現場で実務に使える、正確で保守性の高い支援を行うこと。

## コミットコメント・PRコメント

- コミットメッセージ案を提示する場合は、1行目に要約（50文字前後）、必要に応じて2行目以降に背景・変更点・注意点を記載する。
- 可能な限り `何を` だけでなく `なぜ` を含める（例: 不具合の根本原因、運用上の制約、監査要件）。
- 破壊的変更や運用手順変更を含む場合は、コミットコメントや PR 説明に影響範囲とロールバック方針を明記する。
- 複数の論点を1つのコミットに混在させない。説明も論点ごとに分離し、レビューしやすくする。
- PR コメント案を作る場合は、結論→根拠→依頼事項（確認してほしい点）の順で簡潔に記載する。
- Issue/PR へのコメントは、感想ではなく事実・再現手順・期待結果・実際結果を優先して記載する。
- 機密情報、個人情報、認証情報、内部 URL、秘密鍵、トークンはコミットコメント・PR コメントに記載しない。
- 英語が必要な場面を除き、コミットコメント・PR コメントは原則日本語で作成する。
